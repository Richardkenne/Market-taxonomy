12 Delivery / Operating Models

The finite set of structural patterns through which value reaches the user — independent of market, business system, or technology era.

The 12 canonical delivery models

01 — SaaS

SaaS (Cloud Software)

Multi-tenant software accessed via the web. No installation, continuous updates, shared infrastructure. Examples: Salesforce, Notion, Slack.

02 — API-First

API-First Service

Value consumed programmatically — UI optional. Built to be integrated, not used directly. Examples: Stripe, Twilio, OpenAI API.

03 — On-Premise

On-Premise Software

Installed and operated in the customer's own environment. Maximum control, full IT responsibility. Examples: SAP on-prem, Oracle DB, legacy enterprise software.

04 — Managed Service

Managed Service

Software plus human operation on behalf of the client. The provider runs it, the client uses outcomes. Examples: managed security, cloud MSPs, outsourced IT.

05 — Platform / Ecosystem

Platform / Ecosystem Delivery

Core product extended by third-party developers. Value grows through the ecosystem, not just the core. Examples: Apple App Store, Salesforce AppExchange, Android.

06 — Embedded

Embedded Software

Software integrated into hardware or external systems. Invisible to the end user, essential to the product. Examples: automotive firmware, IoT OS, chip software.

07 — Desktop / Local

Desktop / Local Software

Locally installed applications. Purchased once or via licence. Offline-capable. Examples: Adobe Creative Suite, Microsoft Office (perpetual), Final Cut Pro.

08 — Mobile-Native

Mobile-Native App

Primary experience delivered via mobile OS. Designed for touch, notifications, and device sensors. Examples: Instagram, Uber, Revolut.

09 — Data Feed

Data Feed / Data Access

Structured datasets delivered via stream or batch. The product is the data itself, not an interface. Examples: Bloomberg Terminal, financial data APIs, satellite imagery feeds.

10 — Open-Core

Open-Core + Hosted

Open-source core with managed hosting or enterprise layers on top. Community drives adoption; commercial layer captures revenue. Examples: GitLab, HashiCorp, Elastic.

11 — Marketplace Infra

Marketplace Infrastructure

Infrastructure enabling transactions between third parties. The operator provides the rails, not the goods. Examples: AWS Marketplace, Shopify, payment rails.

12 — Hybrid Delivery

Hybrid Delivery

Stable combination of multiple delivery models. Not a shortcut — a structural outcome when real businesses serve diverse access needs. Examples: Microsoft 365 (SaaS + desktop + API), AWS (IaaS + PaaS + marketplace).
